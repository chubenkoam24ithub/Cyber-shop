## Описание проекта

**Cyber‑shop** – это полнофункциональный интернет-магазин электроники (на примере продукции Apple), реализованный по архитектуре клиент-сервер.  
Проект демонстрирует навыки разработки современных веб-приложений с использованием JavaScript-фреймворков, организации REST API, работы с реляционными базами данных и управления состоянием на клиенте.

### Архитектурные особенности

- **Backend**: Node.js + Express. Реализован REST API для работы с товарами, категориями, характеристиками, изображениями.  
- **База данных**: SQLite + Prisma ORM. Модели данных включают товары, типы продуктов, характеристики, изображения, категории.  
- **Frontend**: Vue 3 (Composition API / Options API) с маршрутизацией Vue Router и глобальным хранилищем Vuex. Приложение полностью реактивно, данные кэшируются и синхронизируются через стор.  
- **Стилизация**: CSS без внешних UI-библиотек, адаптивная вёрстка (медиа-запросы для мобильных устройств, ширина 1120px для десктопа).  
- **Функциональность**:
  - Каталог товаров с пагинацией (12 товаров на страницу)
  - Фильтрация по цене, бренду, категории, характеристикам
  - Сортировка по рейтингу, цене, дате поставки
  - Поиск товаров по названию (выпадающие подсказки)
  - Страница товара с подробными характеристиками, отзывами и сопутствующими товарами
  - Корзина с изменением количества, удалением, подсчётом итоговой суммы
  - Избранное (отдельная страница с понравившимися товарами)

## Задание

В рамках дисциплины «Проектный семинар с фреймворками JavaScript» требовалось разработать полноценное клиент-серверное приложение, удовлетворяющее следующим критериям:

1. **Клиентская часть** на одном из современных фреймворков (Vue 3).  
2. **Серверная часть** на Node.js/Express, предоставляющая REST API и работающая с базой данных.  
3. **Структурированная база данных** (не менее трёх связанных таблиц).  
4. **Функционал**: отображение списка товаров, фильтрация/сортировка/поиск, страница детального просмотра, корзина, избранное.  
5. **Система контроля версий** (Git) и хостинг исходного кода на GitHub.  
6. **Развёртывание** (опционально: возможность локального запуска или публикация на платформе типа Render / Vercel / Railway).
